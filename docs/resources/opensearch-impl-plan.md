## 5. Universal UI rules recap (for coder + QA cross-check)

- ONE Status column only (S4). Currently two ‚Äî drop `Processing` column.
- Healthy rows render blank Status. Banned: `OK`, `available`, `running`, `-`, etc.
- Glyph `!`/`~` on Healthy (green) rows only. Never on non-green.
- S1 badge counts INSTANCES with ‚â•1 `!` finding; `~` never bumps.
- Multi-finding on same row ‚Üí top phrase + `(+N-1)` suffix (rule 7).
- Detail view shows EVERY finding ‚Äî Issues phrases via `injectAttentionSection` (rendered as `<glyph> <capitalizedPhrase>`) + EnrichmentFinding (with `Rows`).
- Wave-2 enricher's `Summary` must be short ¬ß4 phrase; concrete facts belong in `Rows`.
