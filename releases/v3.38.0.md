## JSON View — Raw Resource Data

Press `J` (shift+j) on any resource to see its full AWS API response as pretty-printed JSON.

```
  "InstanceId": "i-0abc123def456789",
  "InstanceType": "t3.medium",
  "State": {
    "Code": 16,
    "Name": "running"
  },
  "LaunchTime": "2026-03-15T09:22:41Z",
  ...
```

### Why JSON?

The YAML view (`y`) is great for reading, but when you need to feed data into AWS CLI commands, CloudFormation templates, jq pipelines, or API calls, you need valid JSON. The JSON view marshals the raw AWS SDK struct directly — native booleans, numbers, nulls, and all fields preserved.

### How it works

| Key | Action |
|-----|--------|
| `J` | Open JSON view (from resource list or detail) |
| `c` | Copy raw JSON to clipboard |
| `/` | Search within JSON |
| `w` | Toggle line wrap |
| `t` | Jump to CloudTrail Events |
| `Esc` | Back to previous view |

### Lambda invocation logs fix

CloudWatch `FilterLogEvents` can return empty pages when scanning across log streams. The invocation logs fetcher now paginates through these empty pages instead of stopping at the first one, fixing "No resources found" for invocations whose logs lived in later log stream pages.
