## Direct Resource Navigation — `-c` Flag

Jump straight to any resource list on startup, just like `k9s -c pods`.

```sh
a9s -c ec2                    # EC2 Instances
a9s -p production -c events   # CloudTrail Events in a specific profile
a9s -c s3 --demo              # S3 Buckets in demo mode
a9s -c vpc -r eu-west-1       # VPCs in eu-west-1
```

### How it works

`-c <name>` resolves the resource type via the same lookup used by `:command` mode inside the TUI — short names (`ec2`, `s3`, `rds`) and aliases all work. If the name doesn't match any registered resource type, a9s prints an error and exits immediately (no blank TUI screen).

After the AWS connection is established, a9s emits a `NavigateMsg` that pushes the resource list onto the view stack. `Esc` returns to the main menu as usual.

### Startup guard

If the initial AWS connection is slow and you navigate away from the main menu while waiting (e.g. open help), the `-c` auto-navigation is silently skipped to avoid pushing a view on top of whatever you're doing. The flag is consumed once and never re-fires on profile or region switches.

### Install

```sh
brew upgrade k2m30/a9s/a9s
# or
go install github.com/k2m30/a9s/v3/cmd/a9s@v3.37.0

docker run --rm -it ghcr.io/k2m30/a9s:v3.37.0 --demo
```
