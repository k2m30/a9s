## New Child Views

- **Step Functions → Executions** (`Enter`): Drill into any STANDARD state machine to see its executions — name, status, start/stop dates, and human-friendly duration. Paginated via NextToken, capped at 200 items. EXPRESS state machines show a flash message explaining execution history is unavailable. (#39)
- **SFN Executions → Execution History** (`Enter`): Drill into any execution to see the step-by-step state machine trace — event type, state name, and error/cause details. Events are classified and color-coded: green for succeeded, red for failed/timed-out, yellow for scheduled/started. Paginated, capped at 500 events. (#40)

## Details

### SFN Executions columns
| Column | Source | Description |
|--------|--------|-------------|
| Name | `path: Name` | Execution name |
| Status | `path: Status` | SUCCEEDED, FAILED, RUNNING, TIMED_OUT, ABORTED, PENDING_REDRIVE |
| Start Date | `path: StartDate` | UTC timestamp |
| Stop Date | `path: StopDate` | UTC timestamp (— for running) |
| Duration | `key: duration` | Human-friendly (e.g., "2m 47s", "~15m" for running) |

### SFN Execution History columns
| Column | Source | Description |
|--------|--------|-------------|
| Timestamp | `path: Timestamp` | Event timestamp |
| Event Type | `key: event_type_short` | Humanized (e.g., "Task Failed", "State Entered") |
| State | `key: state_name` | Step/state name (tracked across events) |
| Detail | `key: event_detail` | Error/cause for failures, input/output for others |

### Architecture additions
- `DrillBlockMessage` on `ChildViewDef` — shows user-facing flash message when drill is blocked (backward-compatible: empty = silent skip)
- `CopyField` on `ResourceTypeDef` — child views specify which field `c` copies to clipboard
