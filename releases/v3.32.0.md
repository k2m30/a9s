## EC2 Status Checks — List & Detail Views (#188)

EC2 instances now surface status check health directly in the TUI.

### List View

The STATE column shows inline indicators when checks are unhealthy on running instances:

- `! running` — one or both checks impaired (system or instance)
- `~ running` — checks still initializing

Healthy instances show plain `running` with no indicator (silence means healthy).

### Detail View

A **Status Checks** section appears after the State section when checks are not fully healthy:

```
 Status Checks:
     System:   ok
     Instance: impaired
```

Values are color-coded: green (ok), red (impaired), yellow (initializing), dim (insufficient-data).
The section is omitted entirely when both checks pass or the instance isn't running.

### Under the Hood

- `DescribeInstanceStatus` is called inline during the EC2 fetch (batched in groups of 100)
- Errors degrade gracefully — no indicators shown if the API call fails
- Demo mode includes instances with impaired and initializing states
- 18 new tests covering list indicators, detail injection, and fetcher behavior
