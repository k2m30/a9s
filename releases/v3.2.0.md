## a9s v3.2.0

### New Child Views

- **Lambda → Invocations** -- Press Enter on a Lambda function to see its recent invocations (last 24h, up to 50). Parsed from CloudWatch Logs REPORT lines. Shows timestamp, request ID, status, duration, memory usage, and cold start detection. Sorted newest-first.
- **Lambda Invocations → Log Lines** -- Press Enter on an invocation to see its full log output: START, application logs, END, and REPORT. Color-coded by severity (ERROR=red, WARN=yellow, START/END=dim). This is the "kubectl logs" equivalent for Lambda debugging.

### How it works

Lambda child views are **cross-service** -- they query CloudWatch Logs, not the Lambda API. The invocations view parses REPORT lines to extract metrics. The log lines view filters by RequestId to show exactly what happened during a single invocation.

The 3-level drill chain: **Lambda Functions → Invocations → Log Lines** closes the debugging loop: see which invocations failed, then read the actual error.

### Known Limitations

- ERROR status detection is not yet implemented. Invocations that threw exceptions show as "OK" in the status column. A future release will add a parallel FilterLogEvents call to cross-reference errors by RequestId.
- Lambda Function Code viewer (issue #29) is deferred -- requires a new viewport-based view type.
