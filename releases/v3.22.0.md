## Generalized Reveal System, Horizontal Scroll Fix, Wrap Toggle (#104, #105, #106)

The reveal feature (`x` key) is no longer hardcoded to Secrets Manager. A new registry-based `RevealFetcher` system allows any resource type to register a reveal action. SSM Parameter Store is the first addition — pressing `x` on an SSM parameter fetches and displays its value with `WithDecryption: true`.

### Registry-Based Reveal

- New `resource.RegisterRevealFetcher()` / `resource.GetRevealFetcher()` API mirrors existing fetcher and child-fetcher registries
- `handleReveal` uses `resource.HasRevealFetcher()` instead of checking for `"secrets"` resource type
- Help overlay dynamically shows the `x` (reveal) key only for resource types that have a registered fetcher
- `SecretRevealedMsg` renamed to `ValueRevealedMsg` with backward-compat alias

### SSM Parameter Reveal

- `x` key on SSM Parameter Store list fetches and displays the parameter value
- SecureString parameters are decrypted automatically (`WithDecryption: true`)
- Works in both live and demo mode

### Horizontal Scroll Fix (#105)

- `ScrollRight` now detects columns that were shrunk below their configured width, not just columns that were dropped entirely
- Previously, scrolling stopped too early when the last visible column was compressed rather than hidden

### Wrap Toggle in Reveal View (#106)

- `w` key now toggles line wrap in the reveal view (previously only worked in YAML and detail views)
- Help overlay shows `w` as "wrap toggle" in the reveal ACTIONS section

### Demo Mode

- Reveal works in demo mode for both Secrets Manager and SSM
- Added `GetSecretValue` and `GetParameter` demo transport handlers
