## Smart Main Menu — Resource Availability & Counts (#68)

The main menu now shows live resource counts and greys out empty resource types, so you instantly see what's in your account without drilling into each service.

- **Resource counts**: menu items show "EC2 Instances (12)", "S3 Buckets (7)", etc.
- **Greyed-out empty types**: resources with zero items are visually dimmed
- **Cursor skips empties**: arrow keys jump over greyed-out items for faster navigation
- **Cached availability**: counts persist across views — no redundant API calls
- **Async probes**: availability checks run in the background, menu stays responsive

## Demo Mode Rewrite (#103)

Demo mode now uses a mock AWS transport layer instead of a separate code path. Same Bubble Tea model, same views, same key bindings — just synthetic data underneath.

- `--demo` flag injects mock HTTP transport into real AWS SDK clients
- Removed the old demo pagination engine entirely
- S3 folder drill-down now returns correct prefix-filtered fixtures

## Pagination UX (#68)

- Help overlay shows pagination context (page position, available actions)
- Long availability text truncated cleanly in constrained layouts
- Page cache persists when navigating between views
