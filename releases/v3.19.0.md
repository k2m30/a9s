## Human-Readable Timestamps (#57)

All timestamp columns across every resource type now display clean `YYYY-MM-DD HH:MM` format — no more ISO 8601 noise with `T` separators, seconds, or timezone suffixes.

**Before:** `2026-01-15T09:22:45+00:00`
**After:** `2026-01-15 09:22`

- 37 AWS fetchers standardized to consistent format
- `fieldpath.FormatValue()` updated — `Path:`-based columns also show the clean format
- S3 objects Size column fixed: now shows `72.8 KB` instead of raw byte count `74557`
- Demo mode fixtures aligned with production format
