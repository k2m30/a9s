## CloudTrail Events — AWSService User Column Fix

The `User` column in CloudTrail Events was still empty for `AssumeRole` events generated by AWS services (e.g., EC2 metadata service calling STS to refresh instance profile credentials). These events have `userIdentity.type = "AWSService"` — no `sessionContext` is present, only an `invokedBy` field.

**Fix:** `extractRoleNameFromCTEventJSON` now handles `"AWSService"` identity type and returns the `invokedBy` value (e.g., `ec2.amazonaws.com`) as the user display string.

| Identity type | User column |
|---|---|
| `AssumedRole` / `Role` | Role name (e.g., `AccountAccessRole`) |
| `AWSService` | Invoking service (e.g., `ec2.amazonaws.com`) |
| `IAMUser` | IAM username (already set by SDK, unchanged) |
