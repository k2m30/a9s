## a9s v3.1.0

### New Child Views

- **Log Groups → Log Streams → Log Events** -- 3-level drill-down chain for CloudWatch Logs. Streams sorted by most recent event. Events are color-coded: ERROR/FATAL (red), WARN (yellow), REPORT (green), START/END (dim). Foundation for all cross-service log views.
- **Target Groups → Target Health** -- Press Enter on a Target Group to see health status per target. Shows target ID, port, AZ, health state, reason, and description. Answers "why 502s?" instantly.

### Improvements

- **Human-readable formatting everywhere** -- Byte sizes show as "25.5 GB" not "27356199665" (Log Groups, Lambda code size, DynamoDB table size). Timestamps show as "2025-07-25 16:05" not epoch milliseconds.
- **Narrow screen support** -- Wide columns shrink to fit instead of disappearing entirely. Log event messages visible on any terminal width.
- **Detail view** -- Dynamic key column width auto-sizes to longest field name. No more truncated field names like "Target.AvailabilityZo...".
- **Sort by age** -- Now works for child view fields (last_event, first_event, timestamp).
- **Log stream pagination** -- Capped at 500 results to avoid loading thousands of streams from large log groups.
- **Newline handling** -- Multi-line log messages (stack traces, pip output) no longer break table row rendering.
