## Related Views Complete — All 66 Resource Types (#119, #243, #188)

Every resource type in a9s now has related-resource views. Press `r` on any detail view to see cross-resource relationships — VPCs show their subnets, security groups, NAT gateways; EC2 instances show their EBS volumes, security groups, target groups; Lambda functions show their log groups, KMS keys, and more.

### What's New

**43 new related-view resource types** (completing the remaining 43 after v3.29.0's initial 23), bringing the total to **66/66** — full coverage across compute, containers, databases, storage, networking, security, monitoring, CI/CD, and messaging.

**Zero nil stubs** — all related-view checkers are real implementations that query the resource cache or call AWS APIs. A new build gate (`TestGolden_LiveCheckerCompleteness`) fails CI if any nil checkers are introduced.

**EC2 inline status checks** (#188) — instance status (ok/impaired/initializing) is now merged into resource fields during the fetch itself, replacing the previous background enrichment pattern. The `DescribeInstanceStatus` call respects the 100-ID-per-call API limit.
