## Seven Resources Re-Implemented from Spec + Unified Attention Section

### Related-resource panel accuracy overhaul

Seven resource types were re-implemented from their specs with drill-through regression tests pinning every pivot:

- **Redis** (`redis`) — `ReplicationGroup`-based fetcher with shard-level signals; cluster-mode Redis now surfaces per-shard state (e.g. `shard ng-1: modifying (+2)`) directly in the Status column.
- **DynamoDB** (`ddb`) — Wave-2 phrases, graph-root pivots, and the `ApproximateZero` contract so truncated backup caches render honest counts.
- **DB Instance** (`dbi`) — full spec rewrite with multi-finding visibility; Aurora instances now land correctly in related navigation.
- **DB Cluster** (`dbc`) — §4 phrases, rule-7 `(+N-1)` suffix, dedicated Wave-2 enricher for DocumentDB/Aurora.
- **S3** (`s3`) — end-to-end related panel (every registered pivot returns a real count on the graph-root fixture).
- **SES** (`ses`) — Wave-1 §4 phrases, account-level Wave-2, SES v1 receipt-rule pivots for Lambda/S3; lambda/eb-rule/kinesis ID-format bugs fixed.
- **Backup** (`backup`) — Wave-2 job-state findings via the unified Status column; DynamoDB and S3 backup coverage now honors wildcard (`arn:aws:dynamodb:*:*:table/*`), `NotResources` exclusions, and tag-based selections (#296).

### Unified Attention section in detail view

Detail views now render a single `Attention (N)` block at the top listing every Wave-1 phrase from `Resource.Issues` and every Wave-2 `EnrichmentFinding` with `!` or `~` glyphs. This replaces the per-type sections (`Pending Maintenance`, `Target Health`, `Latest Build`, etc.) with one consistent format across every resource type. Multiple findings on the same instance stay individually visible — S4 shows `<top phrase> (+N)`, S5 lists every entry.

### Navigable fields now land on the actual resource

Highlighting a KMS key ARN, IAM role ARN, S3 bucket, ECS cluster, or Lambda ARN in a detail view and pressing enter now opens the correct resource. Previously the ARN format mismatch produced empty landing lists. A central `NavIDFromValue` extractor registry normalizes these into bare IDs at navigation time.

### Session-scoped SES cache invalidation

The SES `DescribeActiveReceiptRuleSet` cache is now cleared when switching profile or region, preventing stale Lambda/S3 pivot counts on the related panel after `Ctrl+R` or profile switch.

### Drill-through regression harness

A table-driven integration test (`TestScenario_RelatedDrillThrough_All` + `TestScenario_NavigableFieldDrillThrough_All`) pins every registered related-panel pivot and every navigable field across nine graph-root fixtures. Adding a new resource is one row in `drillThroughFixtures` — no new test function needed.
