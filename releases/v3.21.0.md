## Status Color Expansion — 52 Mappings Across 16 Resource Types (#61)

Row coloring now covers every status/state value across all resource types that have one. Previously only 20 statuses were mapped — now there are 52 direct mappings plus pattern-based matching for CloudFormation.

### New Color Mappings

| Color | New Statuses |
|-------|-------------|
| Green | healthy, ok, issued, deployed, enabled, green, success |
| Red | unhealthy, unavailable, alarm, expired, revoked, rejected, pendingdeletion, red |
| Yellow | draining, initial, insufficient_data, pending_validation, inprogress, healing, rebooting_broker, maintenance, rebooting, resizing, pendingimport, pendingacceptance, yellow, temporary_failure |
| Dim | unused, disabled, inactive, grey, not_started, paused |

### CloudFormation Pattern Matching

CloudFormation stacks have compound status values like `CREATE_COMPLETE` and `UPDATE_ROLLBACK_IN_PROGRESS`. Instead of listing every permutation, a9s now matches suffixes:

- `*_COMPLETE` → green (operation succeeded)
- `*_IN_PROGRESS` → yellow (operation running)
- `*_FAILED` → red (operation failed)

Special case: `ROLLBACK_COMPLETE` → red (the rollback succeeded, but it means the original operation failed).

### Fetcher Improvements

- **SES**: rows now colored by verification status (SUCCESS → green, PENDING → yellow, FAILED → red)
- **CloudFront**: disabled distributions show dim/grey instead of deployed green
- **Elastic Beanstalk**: row color based on Health (Green/Yellow/Red/Grey) instead of operational status
