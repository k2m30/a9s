## Paste Support in Filter and Search + CloudTrail User Fix

### Paste in Filter Mode (Bug 1)

Pressing **Cmd+V** or **ctrl+V** while the filter bar was active (`/`) had no effect — the pasted text appeared in the bar but the list was never filtered, and pressing Enter didn't help.

**Fix:** Both bracketed paste (Cmd+V via terminal) and ctrl+V (clipboard read via textinput) are now correctly routed to the filter input and applied immediately.

### Paste in Search Mode (Bug 2)

Pressing **Cmd+V** or **ctrl+V** while searching in a detail or YAML view (`/` search) also had no effect — the search query was not updated.

**Fix:** Both paste paths (bracketed paste and ctrl+V) now reach the `SearchModel` and append the clipboard content to the search query, with matches recomputed immediately.

### CloudTrail Events — Empty User Column (Bug 3)

The `User` column in the CloudTrail Events list was blank for all `AssumeRole` events (e.g., EC2 instance roles, Lambda execution roles, cross-account access). Only IAM user-initiated events populate the SDK's `Username` field.

**Fix:** When `Username` is nil, the role name from `userIdentity.sessionContext.sessionIssuer.userName` (already extracted for related-panel navigation) is used as a fallback. EC2 and Lambda CloudTrail events now show the role name in the `User` column.
