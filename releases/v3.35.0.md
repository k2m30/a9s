## CloudTrail Events for all resource types + `t` key shortcut

### CloudTrail Events everywhere (#247)

Every resource type now shows "CloudTrail Events" in the right-column related panel — not just EC2 and IAM Users. Press `t` from any resource list, detail, or YAML view to jump directly to CloudTrail Events filtered by the selected resource.

- **Deterministic per-type filters**: each resource type declares a `CloudTrailKey` that specifies exactly how to query CloudTrail (ResourceName by ID, ARN, or Username). No reflection, no heuristics.
- **ARN-aware filters**: Lambda, RDS, EKS, Secrets Manager, DocumentDB, and SQS use full ARN for CloudTrail lookup. EC2, S3, DynamoDB, CFN use bare name/ID. IAM Users and Roles use Username.
- **`t` key shortcut**: works from resource list, detail view (both column states), and YAML view. Suppressed on ct-events lists, child views, and main menu.
- **Status bar + help**: `t` / `cloudtrail` shown in bottom hints and `?` help overlay for all applicable views.

### Demo mode improvements

- 9 new CloudTrail fixture events covering Lambda, RDS, ECS, DynamoDB, Secrets Manager, EKS, and CloudFormation
- Demo CloudTrail fake supports suffix matching (bare name matches ARN-prefixed events)
- SQS resources now expose `Fields["arn"]` for correct demo-mode CloudTrail filtering
