## Configurable Color Themes

### 11 built-in themes (#22)

a9s now ships with 11 color themes, auto-extracted to `~/.a9s/themes/` on first run:

| Dark | Light |
|------|-------|
| Tokyo Night (default) | Tokyo Night Light |
| Catppuccin Mocha | Catppuccin Latte |
| Dracula | Nord Light |
| Nord | Gruvbox Light |
| Gruvbox Dark | Solarized Light |
| Solarized Dark | |

Set a theme in `~/.a9s/config.yaml`:

```yaml
theme: "dracula.yaml"
```

Or switch at runtime: press `:` and type `theme` to open the theme selector (same UX as `:region`). Selection applies immediately and persists across restarts.

### Custom themes

Copy any built-in YAML file from `~/.a9s/themes/`, edit the colors you want, and point your config at it. Partial themes inherit missing colors from the Tokyo Night Dark default. All 35 color slots are configurable — status indicators, YAML syntax highlighting, search highlights, key hints, overlays, and more.

### Architecture

- **Theme struct**: 35 color fields covering the full UI palette, with `DefaultTheme()`, `ApplyTheme()`, `ThemeFromYAML()`, and `ActiveTheme()` API
- **Persist-before-apply**: `:theme` selection writes to config before applying — if save fails, the change is aborted with an error flash
- **Zero-config backward compatible**: no config = Tokyo Night Dark, identical to previous versions
- **NO_COLOR fix**: empty `NO_COLOR=` now correctly forces monochrome per the [no-color.org](https://no-color.org) spec
- **Security**: theme filenames validated against path traversal (rejects `../`, absolute paths, separators); config directory guard prevents accidental writes to working directory
