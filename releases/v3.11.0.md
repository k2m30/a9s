## New Child Views

- **IAM Roles → Attached Policies** (`Enter`): Drill into a role to see all attached managed and inline policies. Merges two API calls (ListAttachedRolePolicies + ListRolePolicies) into a single list. Managed policies show name and ARN; inline policies show name with em-dash for ARN. AdministratorAccess and PowerUserAccess policies highlighted in red. Inline policies rendered dim. (#46)
- **IAM Groups → Group Members** (`Enter`): Drill into a group to see member users. Shows username, user ID, creation date. Uses GetGroup with pagination. Note: PasswordLastUsed is not available from the GetGroup API — column shows "N/A (not in API)". (#50)
- **ELB Listeners → Listener Rules** (`Enter`): Nested level-2 child view (ELB → Listeners → Rules). Shows routing rules with priority, human-readable conditions summary (path/host/source-ip/header patterns joined with AND), action type, and target. Default rule rendered dim. `c` copies the conditions summary. (#51)

## Details

### Role Policies columns
| Column | Source | Description |
|--------|--------|-------------|
| Policy Name | `key: policy_name` | Policy name (managed or inline) |
| Policy ARN | `key: policy_arn` | Full ARN for managed, "—" for inline |
| Type | `key: policy_type` | "Managed" or "Inline" |

### Group Members columns
| Column | Source | Description |
|--------|--------|-------------|
| User Name | `key: user_name` | IAM user name |
| User ID | `key: user_id` | IAM user ID |
| Created | `key: create_date` | Formatted creation timestamp |
| Password Last Used | `key: password_last_used` | Always "N/A (not in API)" from GetGroup |

### Listener Rules columns
| Column | Source | Description |
|--------|--------|-------------|
| Priority | `key: priority` | Rule priority (1, 2, ..., "default") |
| Conditions | `key: conditions_summary` | Human-readable: "path: /api/*", "host: x.com", joined with AND |
| Action | `key: action_type` | forward, redirect, fixed-response |
| Target | `key: action_target` | Target group name, redirect URL, or status code |
