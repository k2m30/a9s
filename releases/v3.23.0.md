## EBS Volumes, EBS Snapshots, AMIs, CloudTrail Event History (#59, #65, #66)

Four new resource types bring a9s from 62 to **66 AWS resource types** across 13 categories.

### New Resource Types

| Resource | Short Name | Category | Pattern |
|----------|-----------|----------|---------|
| EBS Volumes | `ebs` | Compute | B (reuses EC2 client) |
| EBS Snapshots | `ebs-snap` | Compute | B (reuses EC2 client, owner-self filter) |
| AMIs | `ami` | Compute | B (reuses EC2 client, owner-self filter) |
| CloudTrail Events | `ct-events` | Monitoring | Paginated (50 per page) |

### Paginated CloudTrail Events

CloudTrail Events is the first resource type with proper per-page pagination from the start. Instead of fetching all events (which could take minutes and hit API rate limits), the initial load fetches one page of 50 events. Press `M` to load more pages incrementally.

### Truncation Indicator in Main Menu

The availability probe now detects pagination truncation. When a resource type has more items than the first page returned, the main menu shows `(50+)` instead of `(50)`, signaling that more data is available via the `M` key.

### Status Colors

New status mappings for all 4 resource types:
- EBS: `in-use` (green), `available` (yellow), `creating`/`deleting` (blue)
- EBS Snapshots: `completed` (green), `pending` (yellow), `error` (red)
- AMIs: `available` (green), `pending` (yellow), `failed`/`deregistered` (red)
- CloudTrail: `true` (read-only, dimmed), `false` (mutating, normal)

### Demo Mode

All 4 types work in `--demo` mode with realistic synthetic fixtures.
