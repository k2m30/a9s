## CT event detail v2 + demo transport mock + IAM inline policies

### CloudTrail event detail v2 (#245)

Structural redesign of the CT event detail view. New `internal/aws/ctdetail/` package provides Parse, Actor, BuildSections, ExtractTarget, and per-service summarizers (S3, IAM, EC2, generic).

- **13 typed related groups** (was 2): role, iam-user, ec2, s3, s3_objects, lambda, rds, kms, secrets, vpce, sg, ddb, cfn
- **4 self-pivot rows**: CT events by AccessKeyId / Username / EventName / SharedEventId
- **Severity color tiers** in detail view: action row carries the event's severity color
- **ARN navigation**: principal and resource ARNs resolve to navigable fields
- **9 wireframe-aligned fixtures** (Cases A-I) with golden snapshot tests
- **CT events invariant tests**: table-driven over all 18 fixtures covering left-column navigation, right-column checkers, zero-pivot hiding, and end-to-end pivot dispatch

### Test suite strengthening (#248)

- Delete busywork tests, tighten assertions, reveal 4 real bugs
- Lambda invocation log pagination fixed (was silently dropping continuation token)
- Detail RELATED panel fixed at width=60 (layout math corrected with named constants)
- Pagination input-token forwarding verified across all fetchers
- Net: -2,568 lines of test code, +24 behavioral tests

### `:event` alias + tab completion

- `event` / `events` added as recommended aliases for `ct-events`
- Tab in command mode now cycles through matching candidates instead of doing nothing on ambiguous prefixes
- Main menu shows first alias (`:event`) instead of canonical short name

### Demo mode: typed fakes replace legacy HTTP transport (#014)

All 66 resource types now served by 42 per-service typed fakes (`internal/demo/fakes/`) backed by SDK-typed fixture data (`internal/demo/fixtures/`). The legacy HTTP transport, category-based fixture files, parallel demo-related registry, and all `demoMode` branches are deleted -- ~34K lines removed.

- **Single fixture location** -- adding a new resource type requires fixture data in one file
- **Contract-based fakes** -- each fake satisfies its aggregate interface; unknown-parent requests return real SDK error types
- **Main menu counts** -- resource counts appear on startup in demo mode
- **Cold-cache test harness** -- `newDemoColdCacheApp(t)` drives real TUI message flow from empty cache

### IAM inline group policies (#250)

Inline policies now appear in the IAM Policies list alongside managed policies. Policy ID column replaced with Type (managed/inline). Related panel for IAM groups counts both attached managed and inline policies. Inline policy detail shows parent group.

### Bug fixes

- CloudTrail navigable fields: AWSService events no longer pollute user/role_name with service principals; AssumedRole events no longer copy role names into user field
- CloudTrail fake honors LookupAttributes filtering
- Lambda fake filters ListEventSourceMappings by EventSourceArn
- Go toolchain bumped 1.26.1 to 1.26.2 (fixes 4 crypto stdlib vulnerabilities)
