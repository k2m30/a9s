## Demo transport mock + IAM inline policies

### Demo mode: typed fakes replace legacy HTTP transport

All 66 resource types now served by 42 per-service typed fakes (`internal/demo/fakes/`) backed by SDK-typed fixture data (`internal/demo/fixtures/`). The legacy HTTP transport, category-based fixture files, parallel demo-related registry, and all `demoMode` branches are deleted — ~34K lines removed.

Key improvements:
- **Single fixture location** — adding a new resource type requires fixture data in one file, no demo-specific wiring
- **Contract-based fakes** — each fake satisfies its aggregate interface from `internal/aws/interfaces.go`; unknown-parent requests return real SDK error types (not empty lists)
- **Main menu counts** — resource counts now appear on startup in demo mode (availability probes run in noCache mode)
- **Cold-cache test harness** — `newDemoColdCacheApp(t)` drives the real TUI message flow from empty cache against fake clients

### IAM inline group policies

Inline policies now appear in the IAM Policies list alongside managed policies. The "Policy ID" column is replaced with "Type" (managed/inline). The related panel for IAM groups counts both attached managed and inline policies. Opening an inline policy's detail shows its parent group in the related panel.

### CloudTrail navigable field fixes

- AWSService events no longer pollute `Fields["user"]` and `Fields["role_name"]` with service principals
- AssumedRole events no longer copy role names into the user navigation field
- CloudTrail fake honors `LookupAttributes` filtering (Username, EventName, EventSource, ResourceName, ResourceType)

### Main menu alias

The command hint on the main menu now shows the first alias (e.g., `:event`) instead of the canonical short name (`:ct-events`) for a more user-friendly experience.

### Other

- Go toolchain bumped from 1.26.1 to 1.26.2 (fixes 4 crypto/x509 + crypto/tls stdlib vulnerabilities)
- Lambda fake filters `ListEventSourceMappings` by `EventSourceArn`
- CI: `ExecuteCommandConfiguration` excluded from verify-readonly grep (false positive)
