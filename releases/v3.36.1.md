## Patch Release — Cache Atomicity, App Context Lifecycle, Error Surfacing

A round of reliability fixes uncovered during a code audit. No feature changes, no breaking changes, no new dependencies.

### Atomic cache writes

`~/.a9s/cache/<profile>--<region>.yaml` is now written atomically. Previously, `cache.Save` used `os.WriteFile` directly, which opens the target file with `O_TRUNC` — a concurrent reader could observe a zero-length file mid-write, and two a9s processes sharing the same cache would race on a deterministic `.tmp` name and silently lose updates.

The fix uses `os.CreateTemp` for a unique per-invocation temp file followed by `os.Rename`, so every Save either succeeds completely or leaves the previous cache untouched.

### Windows profile name safety

`cache.Path` now sanitizes backslash in addition to forward-slash and space. On Windows, a profile name containing `\` could previously create subdirectories from the file stem.

### Reliable app context cancellation

In-flight AWS fetcher goroutines are now cancelled on every exit path, not just clean `QuitMsg` dispatch. `cmd/a9s/main.go` was refactored into `main()` + `runProgram()` so a `defer model.Cancel()` runs before the `os.Exit(1)` error path. The new public `tui.Model.Cancel()` method is a nil-safe wrapper around the internal cancel function.

### Typed error surfacing for ClientsReadyMsg

Previously, `ClientsReadyMsg` with an unexpected `Clients` type would silently fall back to pre-supplied clients or do nothing, hiding programmer errors behind a blank UI. The handler now distinguishes three cases: `nil` (legitimate demo fallback), correct `*awsclient.ServiceClients` (assign), and wrong type (emit `APIErrorMsg` with a clear internal-error message).
