## New Child Views

- **CodePipeline → Pipeline Stages** (`Enter`): Drill into a pipeline to see the current state of all stages and actions. The hierarchical GetPipelineState response is flattened into stage-action rows with visual grouping — stage name shown only on the first action per stage. Row coloring by action status: green (Succeeded), red (Failed), yellow (InProgress), dim (Stopped/not yet reached). `c` copies the external URL (deep link to CodeBuild, CodeDeploy, etc.). (#43)
- **ECR Repositories → Images** (`Enter`): Browse container images in a repository with tags, digest, push timestamp, human-readable size, and vulnerability scan results. Finding severity is formatted as "1C 3H 5M" (Critical/High/Medium). Row coloring: red for CRITICAL findings, yellow for HIGH, dim for untagged. `c` copies the full image URI (account.dkr.ecr.region.amazonaws.com/repo:tag). Paginated, capped at 500 images. (#44)

## Details

### Pipeline Stages columns
| Column | Source | Description |
|--------|--------|-------------|
| Stage | `key: stage_name` | Stage name (blank for 2nd+ action in same stage) |
| Stage Status | `key: stage_status` | InProgress, Succeeded, Failed, Stopped |
| Action | `key: action_name` | Action name within the stage |
| Action Status | `key: action_status` | Per-action execution status |
| Last Changed | `key: last_change_time` | UTC timestamp of last status change |
| External URL | `key: external_url` | Deep link to CodeBuild/CodeDeploy/etc. |

### ECR Images columns
| Column | Source | Description |
|--------|--------|-------------|
| Tag(s) | `key: image_tags` | Comma-separated tags or `<untagged>` |
| Digest | `key: digest_short` | First 12 chars of sha256 digest |
| Pushed At | `key: pushed_at` | UTC push timestamp |
| Size | `key: image_size` | Human-readable (e.g., "85 MB") |
| Scan Status | `key: scan_status` | COMPLETE, IN_PROGRESS, FAILED |
| Findings | `key: finding_counts` | Severity summary (e.g., "1C 3H 5M") |
